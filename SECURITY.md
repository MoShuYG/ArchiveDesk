# Security Policy

## Supported Versions

ArchiveDesk is currently in an early development stage.

At this time, we do not guarantee long-term support for specific versions.
Security fixes, when available, are most likely to appear in the latest development version.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

If you discover a possible vulnerability, please report it privately to the maintainer first.

When reporting, please include as much of the following as possible:

- A clear description of the issue
- Steps to reproduce
- Affected environment
- Possible impact
- Screenshots, logs, or proof of concept if helpful

## Response Expectations

The maintainer will try to acknowledge valid reports within **7 days**.

Because this project is still early-stage, response and fix timelines may vary.
However, security reports will be treated seriously.

## Scope

Security-sensitive areas may include, but are not limited to:

- Local file scanning
- Metadata extraction
- External file opening integrations
- Authentication and session handling
- API endpoints
- Database access

## Disclosure

Please give the maintainer reasonable time to investigate and prepare a fix before making the issue public.

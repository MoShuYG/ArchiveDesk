# Contributing to ArchiveDesk

感谢你关注 ArchiveDesk。

ArchiveDesk 目前还处于**早期开发阶段**。  
现在最需要的是：帮助项目变得更清晰、更稳定、更容易在 Windows 本地场景中使用。

---

## 当前欢迎的贡献

目前尤其欢迎这些贡献：

- Bug 报告
- 文档修正
- 文案改进
- 小范围 UI / 交互优化建议
- 测试补充
- 扫描、搜索、预览相关问题反馈
- Windows 本地使用场景反馈

---

## 当前暂不建议的贡献

为了避免项目方向过早发散，以下类型的改动建议先不要直接提交 PR：

- 大规模重构
- 引入大型新依赖
- 一次性改很多模块
- 未讨论过的重大产品方向变更
- 以跨平台为目标的大改动

如果你想做这类事情，请先开 Issue 讨论。

---

## 报告 Bug

提交 Bug 时，请尽量带上这些信息：

- 使用环境（例如 Windows 10 / Windows 11）
- 发生问题的页面或功能
- 复现步骤
- 实际结果
- 预期结果
- 截图、日志或报错信息（如果有）

Bug 描述越具体，越容易被处理。

---

## 提功能建议

如果你想提新功能，请先开 Issue，说明：

- 你想解决什么问题
- 当前有哪些不方便
- 你希望的行为是什么
- 这个功能更偏向哪一块（扫描 / 搜索 / 预览 / 管理 / 元数据等）

请尽量先讨论，再开始写代码。

---

## 提交 Pull Request

提交 PR 时，请尽量遵守这些原则：

- 一个 PR 只解决一件事
- 改动尽量小、尽量清晰
- 不要顺手重构无关代码
- 如果改了行为或界面，请说明原因
- 如果修了 Bug，请写清楚复现方式和修复思路

---
在提交前，尽量先自己检查：

```bash
npm test
npm run lint
```

如果前端有改动，也请确认前端可以正常启动。
如果改动涉及 Windows 本地文件流程，优先在 Windows 环境下测试。

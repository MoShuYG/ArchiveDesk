# 为 ArchiveDesk 做贡献

感谢你愿意花时间帮助 ArchiveDesk 变得更好。

贡献不一定要从一个大功能开始。报告一次真实使用中遇到的问题、修正一处文档、补充一个测试，或者把某个操作细节打磨得更顺手，都很有价值。

## 可以从哪里开始

- 使用时遇到问题，可以提交 [Issue](https://github.com/MoShuYG/ArchiveDesk/issues)
- 对功能或交互有想法，也可以先在 Issue 里聊聊
- 已经有明确的小改动，可以直接提交 [Pull Request](https://github.com/MoShuYG/ArchiveDesk/pulls)
- 扫描、搜索、预览、Windows 文件操作和文档都是很适合参与的方向

如果你还不确定从哪里下手，可以先看看现有 Issue，找到自己熟悉或感兴趣的问题。

## 报告问题

提交前可以先搜索一下已有 Issue，看看是否已经有人遇到相同问题。如果没有，请尽量提供这些信息：

- Windows 版本和 Node.js 版本
- 出现问题的页面或功能
- 可以重复触发问题的步骤
- 实际结果和你原本期待的结果
- 相关截图、日志或报错信息
- 涉及的文件格式，以及是否只在某个文件上出现

信息不必写得很正式，只要能帮助别人复现问题就可以。

## 提交功能想法

好的功能建议不需要一开始就给出完整方案。说明你遇到了什么不方便、希望解决什么问题，以及理想情况下操作会有什么变化，通常就足够了。

如果想法涉及大规模重构、新增较大的依赖、跨平台改造或产品方向调整，建议先开 Issue 讨论。这样可以在写代码前确认方向，也能避免重复工作。

## 提交 Pull Request

为了让改动更容易检查和合并，建议：

- 一个 PR 集中解决一件事
- 保持改动范围清晰，不顺手重构无关代码
- 在说明中写清楚为什么要改，以及改动后的行为
- 修复 Bug 时附上复现方式和验证结果
- 修改界面时附上截图或简短说明
- 新增或调整行为时，尽量补上相应测试

## 提交前检查

后端改动至少运行：

```bash
npm test
npm run lint
```

前端有改动时，再运行：

```bash
npm run lint:frontend
npm run build:frontend
```

如果改动涉及 Windows 文件选择、扫描、预览或外部程序调用，请尽量在 Windows 10 或 Windows 11 上实际走一遍相关流程。

感谢你的参与。
